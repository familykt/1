## 자바 인터프리터 (번역기)

자바는 플랫폼으로 부터 자유로운 언어이다. 즉, 우리는 자바 인터프리터만 있다면 어느 플랫폼에서나 자바를 실행할 수 있다. 자바 인터프리터는 자바 바이트 코드(.class 파일) 를 각 운영체제가 이해할 수 있는
코드로 변경한다.

여기에서는 자바에서 인터프리터가 무엇인지, 인터프리터의 특징에 대하여, 또 자바 인터프리터가 어떻게 작동하는지 , 마지막으로 컴파일러와는 어떻게 다른지 살펴볼 것이다.

자바에서 인터프리터란 무엇인가? 자바 인터프리터는 JVM을 실행하는 하나의 컴퓨터 프로그램(시스템 소프트웨어)이다. 즉, 프로그램을 읽거나 실행하는 역할을 한다. 소스 프로그램을 읽어서 인스트럭에 따라 소스코드를
번역함으로서 고급 프로그래밍 언어에서 어셈블리 언어(기계어) 로 변환한다.

## 자바 인터프리터는 어떻게 작동하는가 ?

바이트 코드를 기계 코드로 변경하기 위해, 자바 가상 머신(JVM) 에 .class파일을 올리면, JVM은 해당 코드를 자바 인터프리터를 이용하여 기계 코드로 변경한다. JVM은 런타임시에 인터프리터를 사용하고
변환된 코드를 호스트 머신에서 실행한다.

자바 인터프리터는 자바 컴파일러를 이용하여 소스 코드를 자바 바이트 코드로 변환한다. 같은 방식으로 자바 인터프리터는 바이트 코드를 기계가 이해할 수 있는 형식, 예를 들어 기계 코드 등으로 변환하거나 번역한 뒤,
그 기계 코드로 운영체제에서 동작한다.

어느 시스템이든 JVM이 설치가 되어있다면, 해당 플랫폼에서는 JVM을 이용할 수 있다는 뜻이다. 해당 플랫폼에서는 자바 런타임 시스템의 모든 작업을 실행할 수 있으며, 자바 class 파일을 읽어서 바이트 코드를
번역한다.

구글 크롬, 넷스케이프와 같은 브라우저는 자바 인터프리터를 포함하고 있는 좋은 예이다. 즉, 자바를 사용할 수 있는 브라우저로, 브라우저 내에서 애플릿을 실행하는데 사용되었다. 
인터프리터 역시 최적화된 컴파일러 역할을 수행하여, 자바 바이트 코드를 네이티브 기계 인스트럭션으로 바꾸는 동적 또는 just-in-time(JIT) 컴파일을 실행한다.

인터프리터가 어떻게 자바 프로그램을 읽어오는 지 살펴보자.

우선, 클래스명에 따라 자바 커맨드를 사용함으로써 클래스를 지정하거나, 또는 인터프리터를 위한 옵션 또는 필요하다면, 커맨드라인 전달 인자등을 지정할 수 있다.
클래스를 불러오기 위해 다음과 같은 커맨드를 이용한다.

`% java [interpreter options] class name [arguments]   `

위의 코드에서, 클래스명은 반드시 패키지명이 있다면 포함하여야 한다. 클래스명 뒤에 .class 확장자를 쓸 수 없다.
예를 들어 

`java Product   
java com.javatoint.product.Mobile`

첫번째 커맨드에서 Product 는 클래스명이다.
두번쨰 커맨드에서 com.javatoint.product는 Mobile 클래스가 저장되어 있는 패키지 명이다.

한번 클래스를 불러오면, 자바는 규칙에 따라 main 메소드가 포함된 클래스를 찾는다. JVM이 main 메소드를 찾으면, 인터프리터는 그것을 읽어들이며 어플리케이션을 실행한다. main 메소드를 실행하고 나서 추가적인 쓰레드와 참조하고 있는 다른 메소드등을 실행한다.

## 인터프리터의 특징

인터프리터는 런타임시에 소스 코드를 한줄씩 기계어로 변환한다. 

- 인터프리터는 중간 기계어를 생성하지 않는다.
- 각 라인의 에러는 한줄씩 표시된다.
- 컴파일러에 비해 프로그램 실행 속도가 느리다.
- 프로그램을 분석하고 처리하는 데에 더 적은 양의 시간이 소요 된다.

## 인터프리터와 컴파일러의 차이점


다음 표를 통해 인터프리터와 컴파일러의 주요 차이점을 정리하였다.

| 인터프리터                                            | 컴파일러                                                  |
|----------------------------------------------------|---------------------------------------------------------|
| 인스트럭션에 따라 코드를 번역한다.                         | 전체 프로그램을 한번에 변환                                     |
| 실행이 느리다.                                        | 실행이 빠르다.                                              |
| 컴파일 시간이 적다.                                    | 컴파일 시간이 더 걸린다.                                       |
| 중간 객체 코드를 생성하지 않는다.                         | 중간 객체 코드를 생성한다.                                     |
| 에러가 발견될 때 까지는 컴파일을 수행                       | 컴파일을 끝내고 나서 에러코드를 출력한다.                         |
| Python, PHP, Ruby, Perl 등이 인터프리터               | Java, C++, Scala, C 에서 컴파일러를 사용.                     |
