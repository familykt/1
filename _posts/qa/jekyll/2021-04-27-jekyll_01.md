#### 1. <a id="title_01">들어가며...</a>

필자는 현재 갓 1년이 되지 않은 풋내기 QA 이며, 아직 활발하게 여러 가지를 배우고 있는 중이다. 가뜩이나 아는 것이 없는 데 그나마도 금방 잊어버리기 일쑤라 블로그를 통해 기록을 하기로 마음 먹었다. 이전에
블루호스트를 통하여 워드프레스 블로그를 시도해 본적이 있으나, 결국 얼마 가지 못하고 그만두고 말았다. 내 습관이 좋지 못했던 탓도 있지만, 워드프레스가 내게 맞지 않은 탓도 있었다. 다양한 플러그 인을 통해 비교적
쉽게 여러가지 화려한 설정을 할 수 있었지만, 그만큼 무거워서 글하나 쓰고 확인하기 까지 너무 오래걸렸다. 그에 반해 깃헙 페이지의 경우, MD파일을 편집함으로써 바로바로 확인 및 배포할 수 있는 점이 굉장히
좋았다. ( 심지어 무료..)
현재는 내용은 물론 껍데기도 그닥인 블로그 이지만 하루하루 성장해 나갈 것이라 믿는다.

#### 2. <a id="title_02">필요한 준비</a>

지킬 블로그를 생성하기 위해서는 다음의 두 프로그램이 필요하다.
바로 [루비(Ruby)](https://www.ruby-lang.org/en/)와 [루비 잼(Ruby Gem)](https://rubygems.org/)이다. 루비는 지킬이 쓰여진 프로그래밍 언어이며, 또한 잼은
루비의 패키지 매니저이다. 지킬 블로그를 설치하고 다루기 위해서는 이 두가지를 먼저 컴퓨터에 설치해야 한다. 맥 사용자의 경우에는 루비가 미리 설치되어 있을 수 있으며, 다음 명령어를 통해 확인 할 수 있다.

```
ruby -version
```

또는

```
ruby -v
```

결과로서, 다음과 같이 현재 설치된 루비의 버전이 표시된다. 지킬은 최소 2.4.0 이상이 요구된다.

```
ruby 2.7.0p0
```

같은 방법으로 잼 매니저 또한 설치되어 있는 지 확인 할 수 있다.

```
gem -v
```

#### 3. <a id="title_03">지킬 설치</a>

루비와 잼의 설치 여부를 확인 했으면 이제 지킬을 설치할 차례이다. 다음의 명령어를 통해 설치할 수 있다.

```
gem install jekyll bundler
```

잼 매니저에 의해 자동으로 지킬이 설치되며 위에서 했던 방식으로 설치가 완료되었는지 확인 할 수 있다.

```
jekyll -v
```

정상적으로 설치가 완료되었다면 다음과 같이 지킬 버전이 표시될 것이다.

```
jekyll 4.2.0
```

#### 4. <a id="title_04"> 블로그 생성 </a>

지킬을 정상적으로 설치했다면 다음의 간단한 명령어와 함께 블로그를 생성할 수 있다.

```
jekyll new [폴더명]
```

명령어를 입력하면 블로그가 자동으로 생성되며, 설치가 완료된 후 해당 폴더로 들어가면, 기본 폴더 및 파일이 생성된 것을 확인 할 수 있다. 간단하지 않은가?

<br>
간단하게 폴더를 살펴보자<br>
<ul>
<li>_posts : 포스트가 저장되는 폴더. 설치된 직후에는 예제 파일이 하나 저장되어 있을 것이다. </li>
<li>_site : 최종 산출물이 저장되는 폴더로 서버에 올라갈 정적 파일이 여기에 저장된다. </li>
<li>_config.yml : 블로그 설정을 저장하기 위한 파일 </li>
<li>Gemfile : 의존 관계에 있는 라이브러리를 설치하기 위한 파일</li>
<li>그 외에도 index.md나 about.md와 같은 페이지 파일 등이 있다.  </li>
</ul>

#### 5.<a id="title_05">로컬 서버 실행</a>

이번에는 설치한 블로그를 서버를 통해 띄어보도록 하자. 우선, 커맨드를 통해 해당 폴더로 이동해야 한다. 그리고 다음 명령을 입력한다.

```
bundle exec jekyll serve
```

다음과 같이 로컬 호스트 주소가 생성되면 성공! 해당 주소로 접속하면 기본 설정이 완료된 블로그를 확인할 수 있다.

```
Server address: http://127.0.0.1:4000
```
