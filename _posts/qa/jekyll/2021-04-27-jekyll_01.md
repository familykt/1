#### 1. <a id="title_01">들어가며...</a>
필자는 현재 갓 1년이 되지 않은 풋내기 QA 이며, 아직 활발하게 여러 가지를 배우고 있는 중이다.
가뜩이나 아는 것이 없는 데 그나마도 금방 잊어버리기 일쑤라 블로그를 통해 기록을 하기로 마음 먹었다.
이전에 블루호스트를 통하여 워드프레스 블로그를 시도해 본적이 있으나, 결국 얼마 가지 못하고 그만두고 말았다.
내 습관이 좋지 못했던 탓도 있지만, 워드프레스가 내게 맞지 않은 탓도 있었다.
다양한 플러그 인을 통해 비교적 쉽게 여러가지 화려한 설정을 할 수 있었지만, 그만큼 무거워서 글하나 쓰고 확인하기 까지 너무 오래걸렸다.
그에 반해 깃헙 페이지의 경우, MD파일을 편집함으로써 바로바로 확인 및 배포할 수 있는 점이 굉장히 좋았다. ( 심지어 무료..)
현재는 내용은 물론 껍데기도 그닥인 블로그 이지만 하루하루 성장해 나갈 것이라 믿는다.
#### 2. <a id="title_02">필요한 준비</a>
지킬 블로그를 생성하기 위해서는 다음의 두 프로그램이 필요하다.
바로 [루비(Ruby)](https://www.ruby-lang.org/en/)와 [루비 잼(Ruby Gem)](https://rubygems.org/)이다.
루비는 지킬이 쓰여진 프로그래밍 언어이며, 또한 잼은 루비의 패키지 매니저이다.
지킬 블로그를 설치하고 다루기 위해서는 이 두가지를 먼저 컴퓨터에 설치해야 한다.
맥 사용자의 경우에는 루비가 미리 설치되어 있을 수 있으며, 다음 명령어를 통해 확인 할 수 있다.
```
ruby -version
```
또는
```
ruby -v
```
결과로서, 다음과 같이 현재 설치된 루비의 버전이 표시된다. 지킬은 최소 2.4.0 이상이 요구된다.
```
ruby 2.7.0p0
```
같은 방법으로 잼 매니저 또한 설치되어 있는 지 확인 할 수 있다.
```
gem -v
```
#### 3. <a id="title_03">지킬 설치</a>
루비와 잼의 설치 여부를 확인 했으면 이제 지킬을 설치할 차례이다.
다음의 명령어를 통해 설치할 수 있다.
```
gem install jekyll bundler
```
잼 매니저에 의해 자동으로 지킬이 설치되며 위에서 했던 방식으로 설치가 완료되었는지 확인 할 수 있다.
```
jekyll -v
```
정상적으로 설치가 완료되었다면 다음과 같이 지킬 버전이 표시될 것이다.
```
jekyll 4.2.0
```
